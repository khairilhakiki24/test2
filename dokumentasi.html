KasirSync POS Architecture

Single File Progressive Web App (Offline First)

1 Arsitektur UI / UX Interaktif

  • Mobile Swipe-to-Close Menggunakan implementasi Event Listener Native (touchstart, touchmove, touchend) agar panel keranjang di Mobile dapat ditutup dengan cara di-*swipe* ke bawah layaknya aplikasi iOS / Android native sejati. Mengurangi kekakuan UI statis.
  • Auto-Close & Reflow UX Proses *checkout* langsung menutup keranjang dan popup pembayaran, namun **tetap menampilkan Pop-up Kembalian** untuk memberikan umpan balik (feedback) visual yang penting bagi kasir sebelum mencetak struk.
  • 100% TailwindCSS Seluruh komponen responsif menggunakan Class dasar Tailwind, dibantu dengan CSS spesifik (@media print, Safe Area Inset) guna menyempurnakan bentuk layout.

2 Fitur Komersial & Hardware

🖨 Cetak Thermal Adaptif

Sistem cetak dibungkus pada @media print yang akan menghidupkan elemen tersembunyi berformat struk Thermal (Max 80mm) yang rapi, berpusat, dan *monospaced* menggunakan window.print() tanpa merusak layar aplikasi.

⚡ Cetak Otomatis

Pengguna dapat mengaktifkan Toggle "Cetak Struk Otomatis" di pengaturan agar *browser print dialog* langsung muncul tanpa menutup modal kembalian setelah transaksi.

📤 Export-Import SheetJS

Backup seluruh data transaksi dan produk ke format `.xlsx`, atau import ribuan data produk secara instan ke dalam *IndexedDB* secara offline.

📲 Bagikan via API Native

Integrasi html2canvas yang men-snapshot kode HTML Struk, lalu melemparkannya ke navigator.share() untuk dibagikan ke media sosial (WhatsApp) sebagai nota elektronik.
